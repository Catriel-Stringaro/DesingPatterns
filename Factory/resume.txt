FACTORY PATTERN
===============

The Factory pattern centralizes object creation by hiding the constructor
and exposing named static methods instead. This makes the creation intent
clear and restricts how objects can be instantiated.

SAMPLE 1 — Static Factory Methods (User)
  The User class has a private constructor. The nested Factory class
  exposes named methods (CreateWithDefaultCountry, CreateWithDefaultEmail)
  that pre-fill certain fields, so the caller only provides what's relevant.

SAMPLE 2 — Factory Method (PizzaStore)
  PizzaStore defines an abstract CreatePizza() method. Each subclass
  (NYPizzaStore, FLPizzaStore) overrides it to decide which concrete
  pizza to instantiate. The creation logic is delegated to the subclass.

  also use reflection to create instanse: "(Pizza)Activator.
            CreateInstance(Type.GetType($"FL{Enum.GetName(typeof(TypeOfPizza), type)}Pizza"));

KEY DIFFERENCES BETWEEN BOTH SAMPLES
  Sample 1 → creation lives inside the class itself (nested static class).
  Sample 2 → creation is delegated to subclasses via method override.
  Both are variations of the Factory pattern.
